# 🔍 检查步骤：为什么数据没有写入 MongoDB

## ⚠️ 最重要：后端服务器必须运行！

**数据无法写入 MongoDB 的最常见原因是：后端服务器没有运行。**

---

## 📋 快速检查清单

### 1️⃣ 后端服务器是否在运行？

**打开新的 PowerShell 或 CMD 窗口：**

```bash
cd server
npm install
npm start
```

**必须看到这些信息：**
```
✅ Successfully connected to MongoDB!
📊 Database: dwellsecure
🚀 Server running on port 3000
📍 Health check: http://localhost:3000/health
```

**⚠️ 保持这个窗口打开！** 如果关闭，服务器停止，数据无法保存。

---

### 2️⃣ 测试 MongoDB 连接

**在 server 目录运行：**

```bash
cd server
node test-connection.js
```

**应该看到：**
```
✅ Successfully connected to MongoDB!
📝 Shutoffs in database: X
```

---

### 3️⃣ 测试 API 是否工作

**在浏览器打开：**
```
http://localhost:3000/health
```

**应该看到：**
```json
{"status":"ok","db":"connected"}
```

**如果看不到：** 服务器没有运行。

---

### 4️⃣ 检查应用控制台

**当应用启动时，查看控制台日志：**

**✅ 好的（已连接）：**
```
[API] Platform: android, API URL: http://10.0.2.2:3000
[API] Health check result: CONNECTED ✅
[App] API initialization complete. Available: true
```

**❌ 坏的（未连接）：**
```
[API] Health check failed
[App] API initialization failed
```

---

### 5️⃣ 保存数据时检查日志

**当你保存 shutoff 时，查看应用控制台：**

**✅ 好的（正在保存到 MongoDB）：**
```
[Storage] Saving shutoff: shutoff-123
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[API] POST /api/shutoffs
[API] Request successful: /api/shutoffs
[Storage] ✅ Successfully saved to MongoDB
```

**❌ 坏的（只保存到 AsyncStorage）：**
```
[Storage] API available: false
[Storage] API not available, using AsyncStorage only
```

---

### 6️⃣ 检查服务器终端

**当你保存数据时，服务器终端应该显示：**
```
[API] POST /api/shutoffs - Created shutoff: shutoff-123
```

**如果看不到：** 请求没有到达服务器。

---

## 🚨 常见问题解决

### 问题 1: 服务器没有运行

**症状：** 应用显示 "API not available"

**解决：**
1. 打开新终端
2. `cd server`
3. `npm start`
4. 保持终端打开！

---

### 问题 2: 使用真实设备

**症状：** 网络请求失败

**解决：**
1. 找到电脑 IP：`ipconfig` → "IPv4 地址"
2. 创建 `.env` 文件（项目根目录）：
   ```
   EXPO_PUBLIC_API_URL=http://你的IP:3000
   ```
3. 重启 Expo

---

### 问题 3: MongoDB 连接失败

**症状：** 服务器显示连接错误

**解决：**
1. 运行 `node server/test-connection.js` 测试连接
2. 检查 MongoDB Atlas 网络访问设置
3. 确认连接字符串正确

---
