# 诊断连接问题

## 当前问题

日志显示：
```
WARN  [Storage] API fetch failed, falling back to AsyncStorage: Network request failed
LOG  [Storage] API available: false
WARN  [Storage] API not available, using AsyncStorage only
```

**这意味着：前端无法连接到后端服务器，数据只保存在本地，没有同步到 MongoDB。**

## 诊断步骤

### 步骤 1: 检查后端服务器是否运行

在 `server` 目录下运行：
```bash
cd server
npm start
```

**应该看到：**
```
✅ MongoDB connection established and verified!
🚀 Server running on port 3000
```

**如果没有看到这些消息：**
- 服务器没有启动
- 需要先启动服务器

### 步骤 2: 检查 MongoDB 连接

在服务器终端中，应该看到：
```
✅ MongoDB client connected
✅ Ping successful - database is accessible
✅ MongoDB connection established and verified!
```

**如果看到错误：**
- MongoDB Atlas 集群可能是 Paused 状态
- 网络连接问题
- IP 地址未添加到白名单

### 步骤 3: 检查前端 API URL

查看应用启动时的日志：
```
[API] Platform: android, API URL: http://10.0.2.2:3000
```

**根据平台：**
- **Android 模拟器**: 应该是 `http://10.0.2.2:3000`
- **iOS 模拟器**: 应该是 `http://localhost:3000`
- **物理设备**: 需要设置 `EXPO_PUBLIC_API_URL` 为你的电脑 IP 地址

### 步骤 4: 测试 API 健康检查

在浏览器中访问（或使用 curl）：
```
http://localhost:3000/health
```

**应该返回：**
```json
{
  "status": "ok",
  "db": "connected",
  "timestamp": "2026-01-27T..."
}
```

**如果无法访问：**
- 服务器没有运行
- 端口被占用
- 防火墙阻止连接

## 解决方案

### 方案 1: 启动后端服务器

```bash
cd server
npm start
```

保持这个终端窗口打开！

### 方案 2: 检查 MongoDB Atlas

1. 访问 https://cloud.mongodb.com
2. 检查集群状态（应该是 **Active**，不是 Paused）
3. 检查 Network Access（你的 IP 应该在白名单中）

### 方案 3: 检查网络连接

**Android 模拟器：**
- 确保使用 `http://10.0.2.2:3000`（不是 `localhost`）

**物理设备：**
- 需要设置环境变量：
  ```bash
  # 找到你的电脑 IP 地址（Windows: ipconfig, Mac/Linux: ifconfig）
  # 然后在 .env 或 expo 配置中设置：
  EXPO_PUBLIC_API_URL=http://YOUR_COMPUTER_IP:3000
  ```

### 方案 4: 验证连接

启动服务器后，在应用中：
1. 查看应用日志，应该看到：
   ```
   [API] Health check result: CONNECTED ✅
   ```

2. 尝试创建数据，应该看到：
   ```
   [API] → POST /api/shutoffs
   [API] ✓ POST /api/shutoffs → 200 OK
   [Storage] ✅ Successfully saved to MongoDB
   ```

## 快速检查清单

- [ ] 后端服务器正在运行（`npm start`）
- [ ] 服务器显示 "✅ MongoDB connection established"
- [ ] 服务器显示 "🚀 Server running on port 3000"
- [ ] 应用日志显示 `[API] API available: true`
- [ ] 可以访问 `http://localhost:3000/health`
- [ ] MongoDB Atlas 集群是 Active 状态
- [ ] IP 地址已添加到 MongoDB Atlas 白名单

## 常见错误

### "Network request failed"
- **原因**: 前端无法连接到后端
- **解决**: 确保后端服务器正在运行

### "API_UNAVAILABLE"
- **原因**: 健康检查失败
- **解决**: 检查服务器是否运行，MongoDB 是否连接

### "Database not connected"
- **原因**: MongoDB 连接失败
- **解决**: 检查 MongoDB Atlas 状态和网络访问设置

## 验证修复

修复后，应该看到：
```
[API] Health check result: CONNECTED ✅
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[Storage] ✅ Successfully saved to MongoDB
```

而不是：
```
[Storage] API not available, using AsyncStorage only
```
