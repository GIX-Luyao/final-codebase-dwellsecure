# 🔍 问题诊断：数据没有写入 MongoDB

## ⚠️ 最重要：后端服务器必须运行！

**如果后端服务器没有运行，数据只会保存到本地 AsyncStorage，不会保存到 MongoDB。**

---

## 📋 必须完成的步骤

### 步骤 1: 启动后端服务器

**打开新的 PowerShell 或 CMD 窗口：**

```bash
cd server
npm install
npm start
```

**必须看到：**
```
✅ Successfully connected to MongoDB!
📊 Database: dwellsecure
🚀 Server running on port 3000
```

**⚠️ 保持这个窗口打开！** 关闭窗口 = 服务器停止 = 数据无法保存到 MongoDB

---

### 步骤 2: 验证服务器运行

**在浏览器打开：**
```
http://localhost:3000/health
```

**应该看到：**
```json
{"status":"ok","db":"connected"}
```

**如果看不到：** 服务器没有运行，回到步骤 1

---

### 步骤 3: 检查应用中的状态指示器

**在 Shutoffs 列表页面顶部，你会看到：**

- **绿色 "✅ MongoDB 已连接"** = 正常工作，数据会保存到 MongoDB ✅
- **橙色 "⚠️ 仅本地存储 (MongoDB 未连接)"** = 服务器未运行或无法连接 ❌

**如果看到橙色：** 检查步骤 1 和 2

---

### 步骤 4: 保存数据时查看日志

**当你保存 shutoff 时，查看应用控制台：**

**✅ 好的（保存到 MongoDB）：**
```
[Storage] ✅ Successfully saved to MongoDB
[API] Request successful: /api/shutoffs
```

**❌ 坏的（只保存到本地）：**
```
[Storage] API not available, using AsyncStorage only
[Storage] Saving to AsyncStorage...
```

---

### 步骤 5: 检查服务器终端

**当你保存数据时，服务器终端应该显示：**
```
[API] POST /api/shutoffs - Created shutoff: shutoff-123
```

**如果看不到：** 请求没有到达服务器

---

## 🚨 常见问题

### 问题 1: 服务器没有运行

**症状：** 应用显示橙色 "仅本地存储"

**解决：**
1. 打开新终端
2. `cd server`
3. `npm start`
4. 保持终端打开！

---

### 问题 2: 使用真实手机/平板

**症状：** 网络请求失败

**解决：**
1. 找到电脑 IP：`ipconfig` → "IPv4 地址"
2. 创建 `.env` 文件（项目根目录）：
   ```
   EXPO_PUBLIC_API_URL=http://你的IP:3000
   ```
   例如：`EXPO_PUBLIC_API_URL=http://192.168.1.100:3000`
3. 重启 Expo

---

### 问题 3: MongoDB 连接失败

**症状：** 服务器显示 "MongoDB connection error"

**解决：**
1. 运行 `node server/test-connection.js` 测试连接
2. 检查 MongoDB Atlas 网络访问设置
3. 确认连接字符串正确

---

## 🧪 快速测试

### 测试 1: 在浏览器测试（最简单）

```bash
npm start
# 按 'w' 打开浏览器
```

浏览器可以直接使用 `localhost:3000`，最容易验证。

---

### 测试 2: 手动测试 API

**创建测试 shutoff：**
```powershell
Invoke-WebRequest -Uri http://localhost:3000/api/shutoffs -Method POST -ContentType "application/json" -Body '{"id":"test-123","type":"gas","description":"Test","verification_status":"unverified"}'
```

**查看所有 shutoffs：**
```powershell
Invoke-WebRequest -Uri http://localhost:3000/api/shutoffs
```

如果这些命令工作，API 正常，问题在应用连接。

---

## 📊 验证数据

**在 MongoDB Atlas：**
1. 访问 https://cloud.mongodb.com
2. Browse Collections
3. 选择 `dwellsecure` 数据库
4. 查看 `shutoffs` 集合
5. 应该能看到你的文档！
