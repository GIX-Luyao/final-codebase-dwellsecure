# 🚨 立即检查：为什么 MongoDB 没有变化

## ⚠️ 最重要：后端服务器必须运行！

**如果后端服务器没有运行，数据只会保存到本地，不会保存到 MongoDB。**

---

## 📋 必须完成的 3 个步骤

### ✅ 步骤 1: 启动后端服务器

**打开新的 PowerShell 窗口：**

```bash
cd server
npm install
npm start
```

**必须看到：**
```
✅ Successfully connected to MongoDB!
📊 Database: dwellsecure
🚀 Server running on port 3000
```

**⚠️ 保持这个窗口打开！** 关闭 = 服务器停止 = 数据无法保存

---

### ✅ 步骤 2: 检查应用状态指示器

**在 Shutoffs 列表页面顶部，你会看到：**

- **绿色 "✅ MongoDB 已连接"** = 正常 ✅
- **橙色 "⚠️ 仅本地存储"** = 服务器未运行 ❌

**如果看到橙色：** 回到步骤 1，确保服务器运行

---

### ✅ 步骤 3: 保存数据并检查日志

**当你保存 shutoff 时：**

**查看应用控制台（应该看到）：**
```
[Storage] Saving shutoff: shutoff-123
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[API] POST /api/shutoffs
[API] Request successful: /api/shutoffs
[Storage] ✅ Successfully saved to MongoDB
```

**查看服务器终端（应该看到）：**
```
[SERVER] Received POST /api/shutoffs request
[SERVER] Saving to MongoDB collection: shutoffs
[SERVER] ✅ MongoDB operation result: { upsertedCount: 1, ... }
[SERVER] ✅ Verified: Document exists in MongoDB
[SERVER] POST /api/shutoffs - Created shutoff: shutoff-123
```

**如果看不到这些日志：** 服务器没有运行或请求没有到达

## 🚨 最可能的问题

### 问题 1: 服务器没有运行 ⭐⭐⭐

**症状：** 应用显示橙色状态，控制台显示 "API not available"

**解决：**
1. 打开新终端
2. `cd server`
3. `npm start`
4. 保持终端打开！

---

### 问题 2: 使用真实设备

**症状：** 网络请求失败

**解决：**
1. 找到电脑 IP：`ipconfig` → "IPv4 地址"
2. 创建 `.env` 文件（项目根目录）：
   ```
   EXPO_PUBLIC_API_URL=http://你的IP:3000
   ```
3. 重启 Expo
