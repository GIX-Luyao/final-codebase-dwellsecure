# 🔍 诊断：应用数据没有同步到数据库

## ❌ 问题

刷新后没有任何变化，只有最开始 test 中的数据。

**这说明：** 应用中的数据操作没有保存到 MongoDB，只保存到了本地 AsyncStorage。

---

## 📋 诊断步骤

### ✅ 步骤 1: 确认服务器正在运行

**这是最重要的！**

**打开 PowerShell：**

```bash
cd server
npm start
```

**必须看到：**
```
✅ Successfully connected to MongoDB!
🚀 Server running on port 3000
📊 Database status: CONNECTED ✅
```

**⚠️ 保持这个窗口打开！**

**如果服务器没有运行，数据只会保存到本地 AsyncStorage，不会保存到 MongoDB！**

---

### ✅ 步骤 2: 检查应用状态指示器

**在应用中，Shutoffs 列表页面顶部：**

- **绿色 "✅ MongoDB 已连接"** = 正常 ✅
- **橙色 "⚠️ 仅本地存储"** = 服务器未连接 ❌

**如果显示橙色：** 服务器没有运行或无法连接。

---

### ✅ 步骤 3: 添加数据并查看日志

**当你添加 Property 时：**

**查看应用控制台（Expo 终端），应该看到：**
```
[Storage] Saving property: property-123
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[API] POST /api/properties
[API] Request successful: /api/properties
[Storage] ✅ Successfully saved to MongoDB
```

**如果看到：**
```
[Storage] API not available, using AsyncStorage only
[Storage] Saving to AsyncStorage...
```

**说明：** API 连接失败，数据只保存到本地。

---

**查看服务器终端，应该看到：**
```
[HTTP] POST /api/properties
[数字] ========== POST /api/properties ==========
[数字] Database status: CONNECTED ✅
[数字] ✅ VERIFIED: Document exists in MongoDB
[数字] 📊 Total properties in database: X
```

**如果看不到这些日志：** 请求没有到达服务器。

---

### ✅ 步骤 4: 测试 API 健康检查

**在浏览器中打开：**
```
http://localhost:3000/health
```

**应该看到：**
```json
{
  "status": "ok",
  "db": "connected"
}
```

**如果看不到或显示错误：** 服务器没有运行或无法访问。

---

### ✅ 步骤 5: 检查网络连接（如果使用真实设备）

**如果你使用真实设备（不是模拟器）：**

1. **找到你电脑的 IP 地址：**
   ```powershell
   ipconfig
   ```
   查找 "IPv4 地址"（例如：192.168.1.100）

2. **创建 `.env` 文件（项目根目录）：**
   ```
   EXPO_PUBLIC_API_URL=http://你的IP:3000
   ```
   例如：`EXPO_PUBLIC_API_URL=http://192.168.1.100:3000`

3. **重启 Expo：**
   ```bash
   # 停止当前服务器 (Ctrl+C)
   npm start
   ```

## 🚨 最可能的问题

**90% 的情况下，问题是：服务器没有运行！**

**确保：**
1. 服务器正在运行（`cd server && npm start`）
2. 服务器显示 "Database status: CONNECTED ✅"
3. 服务器窗口保持打开

**如果服务器没有运行，数据只会保存到本地 AsyncStorage，不会保存到 MongoDB！**

## 🎯 解决方案

**如果服务器没有运行：**

1. **启动服务器：**
   ```bash
   cd server
   npm start
   ```

2. **保持窗口打开**

3. **重新测试应用**

**如果服务器正在运行，但仍然没有数据：**

1. **检查应用控制台日志**
2. **检查服务器终端日志**
3. **告诉我你看到了什么**

**一旦服务器运行并且应用连接到服务器，所有数据操作都会保存到 MongoDB！**
