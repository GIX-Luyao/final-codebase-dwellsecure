# 诊断：为什么数据没有写入 MongoDB

## 🔍 问题检查清单

### ✅ 步骤 1: 确认后端服务器是否运行

**打开新的终端窗口，运行：**

```bash
cd server
npm install
npm start
```

**必须看到：**
```
✅ Successfully connected to MongoDB!
📊 Database: dwellsecure
🚀 Server running on port 3000
```

**⚠️ 重要：保持这个终端窗口打开！** 如果关闭，服务器就停止了，数据无法保存到 MongoDB。

---

### ✅ 步骤 2: 测试 MongoDB 连接

**在 server 目录下运行：**

```bash
cd server
node test-connection.js
```

**应该看到：**
```
✅ Successfully connected to MongoDB!
📝 Shutoffs in database: X
```

**如果失败：** MongoDB 连接字符串可能有问题。

---

### ✅ 步骤 3: 测试 API 是否可访问

**在浏览器中打开：**
```
http://localhost:3000/health
```

**或者用 PowerShell：**
```powershell
Invoke-WebRequest -Uri http://localhost:3000/health
```

**应该看到：**
```json
{
  "status": "ok",
  "db": "connected"
}
```

**如果看不到：** 服务器没有运行或端口被占用。

---

### ✅ 步骤 4: 检查应用控制台日志

**当应用启动时，查看控制台：**

**✅ 好的（已连接）：**
```
[API] Platform: android, API URL: http://10.0.2.2:3000
[API] Checking health at http://10.0.2.2:3000/health
[API] Health check result: CONNECTED ✅
[App] API initialization complete. Available: true
```

**❌ 坏的（未连接）：**
```
[API] Health check failed, will use AsyncStorage fallback
[App] API initialization failed
```

---

### ✅ 步骤 5: 保存数据时检查日志

**当你保存一个 shutoff 时，查看应用控制台：**

**✅ 好的（正在保存到 MongoDB）：**
```
[Storage] Saving shutoff: shutoff-123
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[API] POST /api/shutoffs
[API] Request successful: /api/shutoffs
[Storage] ✅ Successfully saved to MongoDB
```

**❌ 坏的（只保存到 AsyncStorage）：**
```
[Storage] Saving shutoff: shutoff-123
[Storage] API available: false
[Storage] API not available, using AsyncStorage only
[Storage] Saving to AsyncStorage...
```

**如果看到"坏的"：** API 连接失败，检查步骤 4。

---

### ✅ 步骤 6: 检查服务器终端

**当你保存数据时，服务器终端应该显示：**
```
[API] POST /api/shutoffs - Created shutoff: shutoff-123
```

**如果看不到：** 请求没有到达服务器。检查：
- 服务器是否在运行？
- URL 是否正确？
- 防火墙设置

---

### ✅ 步骤 7: 在 MongoDB Atlas 中验证

1. 访问 https://cloud.mongodb.com
2. 点击 "Browse Collections"
3. 选择 `dwellsecure` 数据库
4. 查看 `shutoffs` 集合
5. 应该能看到你的文档！

---

## 🚨 最常见的问题

### 问题 1: 服务器没有运行

**症状：** 应用控制台显示 "API not available"

**解决：**
```bash
cd server
npm start
```
保持终端打开！

---

### 问题 2: 使用物理设备测试

**症状：** 网络请求失败

**解决：**
1. 找到你电脑的 IP 地址：
   - Windows: `ipconfig` → 查找 "IPv4 地址"
   - Mac/Linux: `ifconfig` → 查找 `inet` 地址

2. 在项目根目录创建 `.env` 文件：
   ```
   EXPO_PUBLIC_API_URL=http://你的IP地址:3000
   ```
   例如：`EXPO_PUBLIC_API_URL=http://192.168.1.100:3000`

3. 重启 Expo：
   ```bash
   # 停止当前服务器 (Ctrl+C)
   npm start
   ```

---

### 问题 3: MongoDB 连接失败

**症状：** 服务器显示 "MongoDB connection error"

**解决：**
1. 检查 MongoDB Atlas 网络访问设置
2. 确保你的 IP 地址被允许访问
3. 验证连接字符串是否正确

---

## 🧪 快速测试

### 测试 1: 在 Web 浏览器中测试

最简单的方法：

```bash
npm start
# 按 'w' 在浏览器中打开
```

浏览器可以直接使用 `localhost:3000`，最容易测试。

---

### 测试 2: 手动测试 API

**创建测试 shutoff：**
```bash
curl -X POST http://localhost:3000/api/shutoffs ^
  -H "Content-Type: application/json" ^
  -d "{\"id\":\"test-123\",\"type\":\"gas\",\"description\":\"Test\",\"verification_status\":\"unverified\"}"
```

**查看所有 shutoffs：**
```bash
curl http://localhost:3000/api/shutoffs
```

如果这些命令工作，API 是正常的，问题在应用连接。

## 🎯 快速修复

**最可能的问题：服务器没有运行！**

确保你有**两个终端窗口**：
1. **终端 1:** `cd server && npm start` (保持运行！)
2. **终端 2:** `npm start` (你的 Expo 应用)

如果服务器没有运行，数据只会保存到 AsyncStorage，不会保存到 MongoDB！
