# 修复连接问题 - 代码改进

## 问题诊断

应用显示 `API available: false`，即使服务器正在运行。可能的原因：

1. **健康检查超时** - 默认没有超时设置，可能卡住
2. **网络错误** - fetch 请求失败但没有详细错误信息
3. **API URL 不正确** - 平台特定的 URL 可能不对
4. **健康检查失败后没有重试** - 一次失败就永久标记为不可用

## 代码改进

### 1. 添加超时机制

健康检查现在有 5 秒超时，避免无限等待。

### 2. 改进错误日志

现在会显示详细的错误信息，包括：
- 错误类型（超时、网络错误等）
- 完整的错误对象
- API URL 信息

### 3. 改进初始化日志

应用启动时会显示：
- API URL
- 健康检查结果
- 初始化状态

## 测试步骤

### 步骤 1: 重启应用

重启 Expo 应用以加载新的代码。

### 步骤 2: 查看启动日志

应该看到：
```
[API] Platform: android, API URL: http://10.0.2.2:3000
[App] Initializing API connection...
[API] Initializing API connection...
[API] Target URL: http://10.0.2.2:3000
[API] Checking health at http://10.0.2.2:3000/health
```

### 步骤 3: 检查健康检查结果

**如果成功：**
```
[API] Health check result: CONNECTED ✅
[API] Initialization complete. Available: true
[App] API initialization complete. Available: true
```

**如果失败：**
```
[API] Health check failed: [错误信息]
[API] Full error: [完整错误对象]
[API] Will use AsyncStorage fallback
```

### 步骤 4: 根据错误信息诊断

**"Request timeout"** - 服务器没有响应
- 检查服务器是否运行
- 检查端口是否正确

**"Network request failed"** - 无法连接到服务器
- Android 模拟器：确保使用 `http://10.0.2.2:3000`
- 物理设备：需要设置 `EXPO_PUBLIC_API_URL`

**"HTTP 503"** - 服务器运行但 MongoDB 未连接
- 检查服务器日志
- 检查 MongoDB Atlas 状态

## 常见问题解决

### 问题 1: Android 模拟器无法连接

**解决方案：**
- 确保 API URL 是 `http://10.0.2.2:3000`（不是 `localhost`）
- 检查服务器是否监听 `0.0.0.0`（不是 `127.0.0.1`）

### 问题 2: 物理设备无法连接

**解决方案：**
1. 找到你的电脑 IP 地址：
   - Windows: `ipconfig` 查看 IPv4 地址
   - Mac/Linux: `ifconfig` 或 `ip addr`
2. 设置环境变量：
   ```bash
   # 在项目根目录创建 .env 文件
   EXPO_PUBLIC_API_URL=http://YOUR_IP:3000
   ```
3. 重启 Expo 应用

### 问题 3: 超时错误

**解决方案：**
- 检查防火墙设置
- 确保服务器端口 3000 没有被占用
- 尝试增加超时时间（在 `apiClient.js` 中修改）
