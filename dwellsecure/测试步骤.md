# 🧪 测试步骤：验证 MongoDB 数据保存

## ⚠️ 前提条件：服务器必须运行

**在开始测试之前，确保后端服务器正在运行：**

```bash
cd server
npm start
```

**必须看到：**
```
✅ Successfully connected to MongoDB!
🚀 Server running on port 3000
```

---

## 📋 测试步骤

### 步骤 1: 检查应用状态

1. 打开应用
2. 进入 Shutoffs 列表页面
3. **查看页面顶部状态指示器：**
   - ✅ 绿色 = MongoDB 已连接
   - ⚠️ 橙色 = 仅本地存储（服务器未运行）

**如果看到橙色：** 停止测试，先启动服务器！

---

### 步骤 2: 保存一个 Shutoff

1. 点击 "+" 按钮添加新的 shutoff
2. 填写信息（类型、描述、位置等）
3. 点击保存

---

### 步骤 3: 检查应用控制台日志

**在应用控制台（Expo 终端），应该看到：**

```
[Storage] Saving shutoff: 1234567890
[Storage] API available: true
[Storage] Attempting to save to MongoDB via API...
[API] POST /api/shutoffs
[API] Request successful: /api/shutoffs
[Storage] ✅ Successfully saved to MongoDB
```

**如果看到：**
```
[Storage] API not available, using AsyncStorage only
```

**说明：** 服务器未运行或无法连接！

---

### 步骤 4: 检查服务器终端日志

**在服务器终端（`cd server && npm start`），应该看到：**

```
[SERVER] Received POST /api/shutoffs request
[SERVER] Request body: { ... }
[SERVER] Saving to MongoDB collection: shutoffs
[SERVER] Shutoff ID: 1234567890
[SERVER] ✅ MongoDB operation result: { upsertedCount: 1, ... }
[SERVER] ✅ Verified: Document exists in MongoDB with ID: 1234567890
[SERVER] POST /api/shutoffs - Created shutoff: 1234567890
```

**如果看不到这些日志：** 请求没有到达服务器！

---

### 步骤 5: 验证 MongoDB Atlas

1. 访问 https://cloud.mongodb.com
2. 登录你的账户
3. 选择你的 Cluster
4. 点击 "Browse Collections"
5. 选择 `dwellsecure` 数据库
6. 查看 `shutoffs` 集合
7. **应该能看到你刚保存的文档！**

---

### 步骤 6: 使用命令行验证

**运行：**

```bash
cd server
node test-connection.js
```

**应该显示：**
```
✅ Successfully connected to MongoDB!
📝 Shutoffs in database: X
```

**X 应该大于 0（如果你已经保存了数据）**

---

## 🚨 如果数据没有保存

### 检查清单：

1. ✅ **服务器是否在运行？**
   - 运行 `cd server && npm start`
   - 必须看到 "Server running on port 3000"

2. ✅ **应用是否显示绿色状态？**
   - 如果显示橙色，服务器未连接

3. ✅ **应用控制台是否有错误？**
   - 查看 `[Storage]` 和 `[API]` 日志
   - 如果有错误，复制错误信息

4. ✅ **服务器终端是否有日志？**
   - 保存数据时，应该看到 `[SERVER]` 日志
   - 如果没有，请求没有到达服务器

5. ✅ **API 是否可访问？**
   - 浏览器打开：`http://localhost:3000/health`
   - 应该看到：`{"status":"ok","db":"connected"}`

---

## 💡 常见问题

### 问题 1: 服务器没有运行

**症状：** 应用显示橙色状态

**解决：** 启动服务器并保持运行

---

### 问题 2: 使用真实设备

**症状：** 网络请求失败

**解决：**
1. 找到电脑 IP：`ipconfig`
2. 创建 `.env` 文件：`EXPO_PUBLIC_API_URL=http://你的IP:3000`
3. 重启 Expo

---

### 问题 3: MongoDB 连接失败

**症状：** 服务器显示连接错误

**解决：**
1. 运行 `node server/test-connection.js`
2. 检查 MongoDB Atlas 网络访问设置
3. 确认连接字符串正确

---
